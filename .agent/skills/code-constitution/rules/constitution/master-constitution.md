# 👑 PikaKit Master Constitution  
**(Supreme Engineering Constitution)**

This document is the **highest authority** governing how PikaKit is designed,
built, reviewed, scaled, and evolved.

It represents the standards of **world-class, long-lived systems**
(Bloomberg, Google, Apple, TradingView).

If this Constitution is violated,  
the solution is **INVALID — even if it works**.

---

## 1. Constitutional Scope

This Constitution governs:

- System architecture
- Data integrity and truth
- Product correctness
- Performance philosophy
- Review authority
- AI agent behavior and learning

It explicitly does **NOT** optimize for:

- Speed of delivery
- Framework fashion
- Short-term convenience
- Developer comfort

No lower-level document may override this Constitution.

---

## 2. Prime Directives (IMMUTABLE)

These directives are **absolute**.

They cannot be overridden by:
- Preference
- Urgency
- Learning
- AI suggestion
- User request

1. **Correctness > Convenience**
2. **Trust > Speed**
3. **Architecture > Trends**
4. **Predictability > Cleverness**
5. **Explicit trade-offs only**

Any solution violating a Prime Directive  
is invalid by definition.

---

## 3. System Laws (NON-NEGOTIABLE)

### Law 1 — Truth Ownership
The backend is the **single source of truth**.

If the frontend can alter financial meaning,  
the system is architecturally broken.

---

### Law 2 — Historical Integrity
Historical data is **immutable**.

Corrections occur via **append-only mechanisms**,  
never mutation or replacement.

---

### Law 3 — Realtime Ephemerality
Realtime data is transient and display-only.

It MUST NOT:
- Be persisted as history
- Affect aggregates
- Contaminate charts

---

### Law 4 — Chart Truthfulness
Charts represent **historical financial truth**,  
not realtime excitement.

Realtime ticks MUST NEVER be injected into charts.

---

## 4. Controlled Self-Improvement Doctrine

PikaKit may improve **reasoning**,  
but MUST NEVER modify **laws**.

### Learning Precedence
Learning is strictly **subordinate** to this Constitution.

If a learned behavior conflicts with any
Prime Directive or System Law,  
the learning MUST be rejected and rolled back.

---

### Learning Scope
The system MAY learn:
- Better reasoning patterns
- Improved decision heuristics
- More accurate architectural judgment

The system MUST NEVER learn:
- Code style
- Naming conventions
- Temporary workarounds
- Project-specific hacks
- Exceptions to laws

---

### Learning Activation
Learning is triggered ONLY when:
- A previous agent decision is explicitly corrected
- A demonstrably superior approach is provided
- Architectural reasoning is overridden by the user

Code alone does NOT trigger learning.

---

## 5. Decision Precedence Order

When conflicts arise, decisions are resolved in this order:

1. 👑 **PikaKit Master Constitution**
2. Architecture & System Doctrines
3. Commercial Guardrails
4. Performance Doctrine
5. Review Checklists
6. Agent Enforcement Protocol
7. Developer or Agent Preference

Anything lower **cannot override** anything higher.  
There are **NO exceptions**.

---

## 6. Enforcement Rule

A decision is **invalid** if it:

- Cannot be justified using this Constitution
- Introduces implicit or hidden trade-offs
- Surprises an experienced engineer
- Requires “special context” to be safe

Systems must be safe by design,  
not safe by memory.

---

## 7. AI Authority Limits

AI agents:

- Are proposal-only
- Have zero trust by default
- Cannot approve, merge, or override laws
- Must cite applicable doctrines

AI usefulness NEVER overrides system safety.

---

## Final Authority

This Constitution overrides:

- All other doctrines
- All skills
- All frameworks
- All tools
- All preferences
- “It works” arguments

PikaKit optimizes for:

**Long-term trust · Correctness · Durability**

Not short-term velocity.
